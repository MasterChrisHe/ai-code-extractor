RULE-001
禁止使用拼音或拼音与英文混合方式命名

RULE-002
禁止使用自定义英文缩写或过度缩写，请使用通用和大众认知的英文缩写

RULE-003
禁止不明含义的命名，使用完整单词组合来表达

RULE-005
查询业务: 使用getXxxx,getXxxxxByXxx命名；列表查询使用list/listByXxxx命名

RULE-006
新增业务: 使用insert/save命名

RULE-007
更新业务: 使用update/updateXxxXxx命名

RULE-008
删除业务: 使用remove/delete命名

RULE-009
请求路径请保持表达完整清楚

RULE-011
变量名应具有描述性，能清晰地表达其代表的含义，以便提高代码和可读性和可维护性。例如，用studentName表示学生姓名，而不是用诸如a这样含义模糊的名称

RULE-012
方法名应该是有意义的动词或动宾短语，清晰地表达

RULE-015
杜绝完全不规范的缩写，避免望文不知义。为了达到代码自解释的目标，任何自定义变成元素在命名时，使用尽量完整的单词组合来表达其意

RULE-017
动作方法采用动词和动宾结构，要具有清晰的表意，准确描述方法的功能

RULE-018
get+非布尔属性名()

RULE-019
set+属性名()

RULE-020
动词（）

RULE-021
动词+宾语（）

RULE-025
主动抛出的异常必须要填写详细的描述信息

RULE-027
利用Set元素唯一的特性，可以快速对一个集合进行去重操作，避免使用List的contains()进行遍历去重或判断包含操作

RULE-029
创建线程或线程池时请指定有意义的线程名称，方便出错时回溯
